# ReguTwin Agentic OS Frontend

## Overview

The ReguTwin Frontend is the operational interface for the entire compliance platform. It serves as the bridge between users and the autonomous AI compliance engine.

The frontend is not just a dashboard. It is a real-time regulatory operations workspace where compliance officers, auditors, IT security teams, risk teams, and administrators monitor regulations, track compliance actions, investigate conflicts, validate implementations, and generate audit evidence.

The goal of the frontend is to transform complex regulatory workflows into a simple, transparent, and actionable user experience.

---

# Frontend Responsibilities

The frontend is responsible for:

* Displaying regulations detected by the Watchman Agent
* Showing AI-generated compliance obligations
* Managing Measurable Action Points (MAPs)
* Visualizing regulatory conflicts
* Monitoring validation results
* Providing audit trails
* Delivering real-time notifications
* Offering executive compliance dashboards

The frontend never performs AI processing directly.

Its role is to consume data generated by:

* Backend APIs
* AI Agents
* Validation Services
* ChromaDB Search Results
* LangGraph Workflows

# Main Modules

Provides a real-time overview of compliance health.

Includes:

* Compliance score
* Active regulations
* Pending MAPs
* Failed validations
* Conflict alerts

This page acts as the command center.

---

## Regulations Module

Displays regulations detected by the system.

Features:

* Regulation listing

* Regulation details

* Obligation extraction

---

## MAP Management Module

Displays compliance tasks generated from regulations.

* Priority management

* Status updates

---

## Conflict Intelligence Module

# State Management Strategy

Global state should be stored using Zustand.

Recommended stores:

* authStore
* userStore
* notificationStore
* dashboardStore
* regulationStore
* mapStore
* validationStore

Server data should be managed using React Query.

---

# API Communication

The frontend communicates only with backend APIs.

Frontend → Backend → AI Layer

Never:

Frontend → OpenAI

Frontend → Gemini

Frontend → ChromaDB

Frontend → LangGraph

All AI communication must remain server-side.

---

# Realtime Features

The platform should support:

* Notification delivery

# Design Philosophy

The interface should feel:

* Enterprise-grade
* Clean
* Professional
* Data-driven
* Minimalistic

Avoid excessive animations.

Users must immediately understand compliance status and risks.

---

# Frontend Success Criteria

A successful frontend should allow a compliance officer to:

1. View a newly detected regulation.
2. Understand AI-generated obligations.
3. Review generated MAPs.
4. Investigate conflicts.
5. Monitor validation results.
6. Access audit evidence.

All without needing to interact with the AI layer directly.

The frontend should make complex compliance workflows appear simple, actionable, and transparent.
